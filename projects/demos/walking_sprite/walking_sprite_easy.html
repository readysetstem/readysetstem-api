Walking Sprite - Easy

In this demo we will be demonstrating a simple animations. This will be acheived by quickly showing different sprites in succession to simulate the appearance of the sprite walking.

Programming

Setting up the sprites

  1. There are 7 sprite files called man1.spr to man7.spr. These files are sprites that show the walking guy and different points in his walk.

    The sprite files are like the individual pages of a flip book. We want to define these pages to the program.

    The first thing we do is tell the program where these sprite files are

  2. Next we list these sprites in the correct order just like putting the pages of a flipbook in the right order. We call this list frames.

Setting up the looping

  1. Next we want to set up an infinite loop using a while followed by True. This tells the program that "while True is True, we want to loop through the indented stuff"
  2. Next we want to loop through all of the frames in our frames list.
    for current_sprite in frames: will go through each sprite in the frames list, in order, and run the code below it. The current sprite that the for loop is on when running the code below it will be the variable current_sprite.

Displaying sprites on the LED matrix

  1. Clear the LED Matrix from the previous sprite that was displayed by using led_matrix.erase()
  2. Now draw our current_sprite onto the LED Matrix by using led_matrix.sprite(). This will tell the LED matrix that we want our current_sprite on the LED matrix.
  3. Drawing the sprite does not actually display the sprite on the LED matrix. To actually display the sprite we must call led_matrix.show()
  4. Now that the current_sprite has been displayed. The program should wait a little bit before attempting to display the next sprite. If we don't the sprites will change too fast to be able to see the animation. (Similar to if you go through a flipbook too fast.) By calling time.sleep(.25), we tell the program to wait .25 or 1/4 of a second before attempting to display the next sprite.

    Try This: Try changing the delay time in time.sleep(.25) and see how if affects the animation!
