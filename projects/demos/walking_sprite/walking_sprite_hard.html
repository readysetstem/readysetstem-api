Programming

Setting up the sprites and variables

  1. There are 7 sprite files called man1.spr to man7.spr. These files are sprites that show the walking guy and different points in his walk.

    The sprite files are like the individual pages of a flip book. We want to define these pages to the program.

    The first thing we do is tell the program where these sprite files are by using led_matrix.LEDSprite() to make sprite variables. And at the same time, put them into a list called frames. Similar to the process done in Easy.

    We can do this efficiently by using a for loop and taking advantage of the fact that all the sprite are labeled manX.spr where X is a number.

    Note: range(1,8) will tell the for loop to loop from 1 to 7 (not encluding 8). i is the variable representing the current number the for loop is on.

    For example:
    for i in range(1,4):
        print(i)

    Would print:
    1
    2
    3

  2. We want to setup a variable called moving_right. This variable will be a boolean (2 values True or False). The variable will be used to help the program know if the sprite is currently walking to the right or to the left. (This is information the program needs to know, to know how to properly draw the sprite)
  3. We then setup a variable called x which will indicate the current horizontal position the sprite is located. (This is another useful piece of information for the program to know how to draw the sprite.)

Setting up the looping

  1. Next we want to set up an infinite loop using a while followed by True. This tells the program that "while True is True, we want to loop through the indented stuff"
  2. Next we want to loop through all of the frames in our frames list.
    for current_sprite in frames: will go through each sprite in the frames list, in order, and run the code below it. The current sprite that the for loop is on when running the code below it will be the variable current_sprite.

Displaying sprites on the LED matrix

  1. Clear the LED Matrix from the previous sprite that was displayed by using led_matrix.erase()
  2. In order to figure out how to draw the next sprite, we need to figure out if the sprite is currently walking left or right. If moving_right == True, we should display the sprite facing the original direction (right). If moving_right == False the sprite is moving left and we should display the sprite facing the other direction by drawing a sprite that has been flipped_vertically.

    In the if statement we are drawing the sprite at either the position (0,0) or (0,2).

    This extra parameter in led_matrix.sprite(), are the coordinates where, with respect to the bottom left corner the LED matrix, the sprite should be displayed.

    For Example:
    led_matrix.sprite(current_sprite, (1,2)) would display the sprite 1 pixel to the right and 2 pixels up from the pixel at the bottom left corner.

  3. Drawing the sprite does not actually display the sprite on the LED matrix. To actually display the sprite we must then call led_matrix.show()
  4. Now we need to either increase our decrease our x variable by 1 depending if the sprite was going left or right. We determine which way it was going with an if moving_right: statement.
  5. We then need to determine if we need to change the moving_right.

    The variable needs to be changed from True to False if the sprite has hit the right side of the LED matrix display. Which is when x > (led_matrix.width() - sprite.width) is True.

    Otherwise the variable needs to be changed from False to True if the sprite has hit the left side of the LED matrix display. Which is when x == 0

  6. Now that the current_sprite has been displayed. The program should wait a little bit before attempting to display the next sprite. If we don't the sprites will change too fast to be able to see the animation. (Similar to if you go through a flipbook too fast.) By calling time.sleep(.25), we tell the program to wait .25 or 1/4 of a second before attempting to display the next sprite.

    Try This: Try changing the delay time in time.sleep(.25) and see how if affects the animation!
