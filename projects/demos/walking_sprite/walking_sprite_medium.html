Walking Sprite - Medium
Jumping Man

In this demo we will be demonstrating a simple animation of a guy walking like in Easy but add the ability to press a button to make the guy jump.

Setup

Need:

  • At least 1 LED matrix
  • A button
  • 2 jumper wires for the button

Steps:

  1. Connect led matrix together by ....... TODO: link to already existing instructions?
  2. Place the button on the breadboard.
  3. Using a jumper wire, connect on side of the button to a GND pin on the Raspberry Pi.
  4. Using the other jumper wire, connect the other side of the button to pin 4 on the Raspberry Pi.

Programming

Setting up the button and sprites

  1. First, we want to set up the button that is connected to port 4 of the Raspberry Pi. We set up a my_button variable that defines the button attached to port 4, by calling button.Button(4)

  2. There are 7 sprite files called man1.spr to man7.spr. These files are sprites that show the walking guy and different points in his walk.

    The sprite files are like the individual pages of a flip book. We want to define these pages to the program.

    The first thing we do is tell the program where these sprite files are by using led_matrix.LEDSprite() to make sprite variables. And at the same time, put them into a list called frames. Similar to the process done in Easy.

    We can do this efficiently by using a for loop and taking advantage of the fact that all the sprite are labeled manX.spr where X is a number.

    Note: range(1,8) will tell the for loop to loop from 1 to 7 (not encluding 8). i is the variable representing the current number the for loop is on.

    For example:
    for i in range(1,4):
        print(i)

    Would print:
    1
    2
    3

Setting up the looping

  1. Next we want to set up an infinite loop using a while followed by True. This tells the program that "while True is True, we want to loop through the indented stuff"
  2. Next we want to loop through all of the frames in our frames list.
    for current_sprite in frames: will go through each sprite in the frames list, in order, and run the code below it. The current sprite that the for loop is on when running the code below it will be the variable current_sprite.

Displaying sprites on the LED matrix

  1. Clear the LED Matrix from the previous sprite that was displayed by using led_matrix.erase()
  2. In order to figure out how to draw the next sprite, we need to figure out if the user has pressed the button. If the user has pressed the button, we should display the sprite higher up off the ground.

    To check, we create an if statement that checks if the user has pressed the button. my_button.is_pressed() will be True if the user has the button pressed down making the if statement to also be true and call the line under it.

    If the statement ends up being False because the user does not have the button pressed down the program will instead run the line under the else:.

    In the if statement we are drawing the sprite at either the position (0,0) or (0,2).

    This extra parameter in led_matrix.sprite(), are the coordinates where, with respect to the bottom left corner the LED matrix, the sprite should be displayed.

    For Example:
    led_matrix.sprite(current_sprite, (1,2)) would display the sprite 1 pixel to the right and 2 pixels up from the pixel at the bottom left corner.

    Try This: Try changing the coordinates of the sprite when the button is pressed, and see what happens when you press the button!

  3. Drawing the sprite does not actually display the sprite on the LED matrix. To actually display the sprite we must then call led_matrix.show()
  4. Now that the current_sprite has been displayed. The program should wait a little bit before attempting to display the next sprite. If we don't the sprites will change too fast to be able to see the animation. (Similar to if you go through a flipbook too fast.) By calling time.sleep(.25), we tell the program to wait .25 or 1/4 of a second before attempting to display the next sprite.

    Try This: Try changing the delay time in time.sleep(.25) and see how if affects the animation!
